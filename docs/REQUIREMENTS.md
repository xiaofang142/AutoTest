## 1. 产品定位

### 1.1 一句话价值

> 让任何人（不需要编程背景）输入一个网址，就能获得一份 AI 生成的、可直接指导修复的测试报告。

### 1.2 核心承诺

| # | 承诺 | 说明 |
|---|------|------|
| 1 | **零脚本** | 不需要写任何测试代码、XPath、选择器 |
| 2 | **零维护** | 页面改版不影响测试（AI 视觉驱动，非 DOM 耦合） |
| 3 | **零配置** | 不需要 API key、不需要数据库就能运行核心功能 |
| 4 | **全程自动化** | 文档理解 → 场景生成 → 执行 → 校验 → 报告，无需人工介入 |

### 1.3 目标用户

| 用户 | 角色 | 核心场景 |
|------|------|----------|
| QA 测试工程师 | 主要用户 | 输入网址和 PRD，一键回归测试 |
| 前端开发 | 快速验证 | 提交代码后自动冒烟测试 |
| 产品经理 | 消费报告 | 查看覆盖度报告和缺陷趋势 |
| AI 编程助手 | 数据消费者 | 通过 MCP 获取缺陷数据直接修复 |

### 1.4 顶层对象

本版本开始，AutoTest 的产品主对象统一为:

**`TestTask`（自动测试任务）**

在需求层，`TestTask` 只表示“一次完整自动测试尝试”的业务对象。

详细对象结构、子对象关系与状态机定义，统一参考:

- [自动测试任务模型设计.md](./自动测试任务模型设计.md)
- [全自动AI测试闭环设计.md](./全自动AI测试闭环设计.md)

## 2. 用户流程

### 2.1 零文档快速测试模式

```
用户输入网址 (https://myapp.com/login)
  │
  ▼
系统自动打开浏览器 → 导航到该网址
  │
  ▼
自动发现页面结构 (标题、按钮、输入框、链接)
  │
  ▼
自动生成基础测试 (页面加载、点击每个按钮、填写输入框)
  │
  ▼
执行每一步 → 截图 → 采集控制台日志 → 记录网络请求
  │
  ▼
四维校验 (UI 渲染错误、JS 异常、API 报错、业务逻辑)
  │
  ▼
输出: 缺陷报告 + 截图证据 + 控制台异常
```

### 2.2 完整文档驱动测试模式

```
用户输入网址 + 上传 PRD/UI 规范/API 文档
  │
  ▼
AI 提取业务规则 → 生成结构化测试场景
  │
  ▼
按场景执行 → 每步截图+日志+网络 → 四维校验
  │
  ▼
输出: 结构化缺陷数据 (MCP) + AI 根因分析 + 修复建议
```

### 2.3 统一闭环阶段

无论是零文档快速模式还是完整文档驱动模式，都必须遵循统一闭环。

阶段名称与详细降级策略，以 [全自动AI测试闭环设计.md](./全自动AI测试闭环设计.md) 为准。

阶段要求:

- 每一阶段都必须有明确输入和输出
- 每一阶段都必须支持状态追踪
- 每一阶段都必须定义失败时的降级策略

## 3. 功能需求

### F-00: 自动测试任务

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F-00.1 | 创建自动测试任务 | P0 | 输入 URL、平台、可选文档，系统创建任务 |
| F-00.2 | 任务阶段跟踪 | P0 | 查看任务当前处于预检/理解/规划/执行/归因等哪个阶段 |
| F-00.3 | 任务详情 | P0 | 汇总输入、蓝图、执行、缺陷与交付结果 |
| F-00.4 | 任务取消 | P1 | 中断正在运行中的任务 |
| F-00.5 | 阻塞反馈 | P1 | 明确显示任务为何 blocked 及需要补充什么 |
| F-00.6 | 任务交付包 | P1 | 输出面向测试、开发者、AI 助手的结果包 |

### F-01: 项目管理

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F-01.1 | 创建项目 | P0 | 输入项目名称、被测系统 URL、可选文档 |
| F-01.2 | 项目列表 | P0 | 查看所有项目及状态 |
| F-01.3 | 项目详情 | P0 | 查看项目配置、文档、执行历史 |
| F-01.4 | 编辑项目 | P1 | 修改项目名称、URL |
| F-01.5 | 删除项目 | P1 | 软删除 |

### F-02: 文档管理

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F-02.1 | 添加文档引用 | P0 | 通过 URL 或上传 .md 文件 |
| F-02.2 | 文档解析 | P0 | 自动拉取内容 → 分块 → 分类 → 提取业务规则 |
| F-02.3 | 查看解析状态 | P0 | 解析进度、规则数量、错误信息 |
| F-02.4 | 规则管理 | P1 | 查看/编辑 AI 提取的规则 |

### F-03: 测试执行

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F-03.1 | 一键执行 | P0 | 创建执行计划并启动 |
| F-03.2 | 浏览器执行 | P0 | 打开被测网站，AI 视觉操作每一步 |
| F-03.3 | 数据采集 | P0 | 每步记录：截图、控制台日志、网络请求、页面状态 |
| F-03.4 | 执行进度 | P1 | 实时推送进度（当前步骤/总步骤） |
| F-03.5 | 取消执行 | P1 | 中断正在执行的测试 |
| F-03.6 | 重试失败步骤 | P2 | 只重试失败的测试步骤 |
| F-03.7 | 自动恢复动作 | P1 | 定位失败、弹窗阻塞、新开页跳转等场景自动恢复 |

### F-04: 四维校验

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F-04.1 | UI 校验 | P0 | 检测页面错误提示、空白页、渲染异常 |
| F-04.2 | 控制台校验 | P0 | 捕获 JS Error、Warning、未捕获异常 |
| F-04.3 | API 校验 | P0 | 检测 4xx/5xx、超时、响应结构异常 |
| F-04.4 | 业务校验 | P1 | URL 跳转、关键文本、页面标题 |

### F-05: 报告与缺陷

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F-05.1 | 执行报告 | P0 | 步骤列表 + 状态 + 截图 |
| F-05.2 | 缺陷列表 | P0 | 按严重程度排序的缺陷清单 |
| F-05.3 | 缺陷详情 | P0 | 证据链：截图 + 日志 + 网络 + AI 分析 |
| F-05.4 | MCP 接口 | P1 | AI 编程工具通过 MCP 消费缺陷数据 |
| F-05.5 | 修复上下文 | P0 | 输出可直接供开发者和 AI 助手消费的 RepairContext |
| F-05.6 | 回归建议 | P1 | 为每个关键缺陷生成最小回归验证入口 |
| F-05.7 | AI 交付结构化输出 | P0 | 面向 AI 助手的 RepairContext + 结构化 JSON |

### F-06: OCR 增强 (NEW)

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F-06.1 | 图片预处理 | P0 | OCR前自适应二值化、去噪、对比度增强、文本纠偏 |
| F-06.2 | 布局区域分析 | P0 | 自动识别 header/main/sidebar/footer/modal 区域 |
| F-06.3 | DOM-OCR 坐标对齐 | P1 | OCR 文本块与 DOM 元素通过 IoU 匹配 |
| F-06.4 | Chain-of-Thought 分析 | P0 | LLM 分步推理: API→Console→UI→Business 逐层判断 |
| F-06.5 | 跨步上下文 | P1 | 前一步分析结果影响后续步骤判断 |
| F-06.6 | 规则引擎降级 | P0 | 无 LLM API Key 时自动降级为规则引擎四维校验 |
| F-06.7 | 项目源码分析 | P0 | 输入代码目录自动扫描路由/组件/API端点, 生成精准测试蓝图 |

## 4. 非功能需求

| ID | 要求 | 指标 |
|----|------|------|
| NF-01 | 零外部依赖运行 | 无 API key、无数据库可执行核心功能 |
| NF-02 | 单步执行时间 | ≤ 10s（含 Playwright 定位 + OCR 识别） |
| NF-03 | 截图采集 | 每步自动截取视口截图 |
| NF-04 | 数据保留 | 执行记录保留在内存（重启即清空） |
| NF-05 | 阶段可追踪 | 每个任务阶段均有明确状态与时间戳 |
| NF-06 | 自动化等级 | 每个任务应输出自动化等级 A0-A5 |

---

## 5. 技术约束

| ID | 约束 | 说明 |
|----|------|------|
| TC-01 | 执行引擎 | Playwright (3-level DOM) + PaddleOCR 本地 OCR |
| TC-02 | 后端框架 | Python FastAPI |
| TC-03 | 前端框架 | Vue 3 + Element Plus |
| TC-04 | AI 服务 | LiteLLM（多模型支持），无 key 时规则引擎降级 |
| TC-05 | 存储 | 内存存储（默认），可切换到 PostgreSQL |
| TC-06 | 状态管理 | 项目/执行/缺陷数据通过 API 管理 |

---

## 6. 不做的事情 (Out of Scope)

- 不提供 CI/CD 集成（仅 API 触发）
- 不提供多用户/权限管理
- 不提供定时执行
- 不提供跨平台报告对比
- 不提供性能测试
- 不提供安全测试
